# Clippings Skill Harvester


Use when processing a folder of saved clippings, source notes, exported articles, or raw notes.

Workflow:
1. Build/update a hash log so unchanged files are skipped.
2. For each new/changed clipping, capture title/path/hash/source metadata.
3. Classify claims, evidence quality, and reusable procedures.
4. Assign a disposition: use-now, route-project, skill-runbook, experiment, source-to-read, research-lead, defer, kill/archive, already-absorbed.
5. Draft skill/runbook patch candidates only for reusable procedures.
6. Record rejected/deferred ideas so they do not recur.
7. Write one dated synthesis report, not one note per clipping unless a source deserves it.
8. Verify paths, hashes, links, and absence of secrets.

Do not create a new skill for every interesting clipping.
