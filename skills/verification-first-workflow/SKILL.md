# Verification First Workflow


Use after file edits, vault changes, script changes, ledgers, or generated reports.

Check path existence, parseable formats, diff cleanliness, links, secrets, schema validity, tests, and source handles. Report exact checks run and exact failures. Never fabricate verification.
