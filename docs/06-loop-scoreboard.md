# Loop Scoreboard

Every recurring loop earns one verdict:

- `keep`: clear signal and acceptable burden.
- `tune`: useful but too noisy/broad.
- `pause`: wait for clearer use case/input.
- `kill`: stop/archive the loop.
- `merge`: combine with another loop.

Score by decision impact, signal quality, review burden, false-positive risk, time saved, artifact production, and token/human attention cost.

Kill or pause when output is rarely reviewed, changes no decision/artifact/rule, duplicates another loop, or creates more review surface than action.
