# Deep Timeline Seed

The starter kit includes a small public-domain historical seed so the vault is not a blank history graph. This is not a substitute for source-grounded historical research. Treat it as scaffolding for learning how to convert historical events into reusable mechanisms, bridge notes, questions, and source queues.

## Purpose

The Deep Timeline layer helps an agent learn long-range patterns without turning every analogy into proof. It should support:

- chronological orientation;
- source-spine building;
- mechanism extraction;
- cross-domain bridge notes;
- falsifiers and rival explanations;
- project applications;
- non-connections where a tempting analogy fails.

## Minimum workflow

```text
historical seed event
-> source spine / reading queue
-> event note
-> mechanism candidates
-> bridge notes / questions
-> application checklist
-> falsifier / non-application boundary
```

## Use rules

- Prefer primary or scholarly sources when promoting a claim.
- Keep date uncertainty visible for ancient history.
- Do not treat resemblance across eras as causation.
- Include rival explanations and counterexamples.
- Promote mechanisms in small batches, not as a giant list.
- Preserve source queues so future agents know what remains weak.
