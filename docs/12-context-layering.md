# Context Layering

Put information where it belongs.

- Persona/system prompt: voice and broad operating rules.
- Memory: stable user/environment facts.
- Skills: repeatable procedures.
- Obsidian: durable source-grounded knowledge.
- Scripts: deterministic repeated transforms/checks.
- Ledgers: structured recurring state.
- Sidecars: derived/rebuildable data.
- Chat: ephemeral workbench.
- Reports: compact summaries with evidence handles.

If it will be stale in a week, do not put it in memory. If it should change future behavior, consider a skill/runbook. If it is source truth, keep it in Obsidian/Git.
