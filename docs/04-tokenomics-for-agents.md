# Tokenomics for Agents

Token efficiency is an attention economy, not just cost cutting.

Four budgets:

1. Context budget: what enters the active prompt.
2. Retrieval budget: what gets searched or loaded.
3. Reasoning budget: what deserves strong models, tools, or reviewers.
4. Review budget: how much human/agent attention the output creates.

Rules:

- Use router skills before heavyweight specialist skills.
- Use Obsidian/files as durable truth, not active chat context.
- Use skills for procedures, not raw knowledge.
- Use memory only for stable facts.
- Use hash logs to avoid rereading unchanged clippings.
- Pre-compress large folders/logs into small maps before LLM reasoning.
- Put evidence in files; send cockpit summaries to chat.
- Kill loops that only produce more review surface.
- Prefer script-only jobs for deterministic fixed-shape alerts.
- Use fresh sessions when topic/tool context changes.
